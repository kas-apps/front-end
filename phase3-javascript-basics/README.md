# Phase 3: JavaScript 基礎編 ⚡

**目標**：プログラミングの基本を理解し、簡単な動的処理が書けるようになる

---

## この Phase で学ぶこと

Phase 1 と 2 では、HTML/CSS でレイアウトとスタイルを作る方法を学んだね！

でも、それだけでは「見せる」ことしかできない。ユーザーのクリックに反応したり、データを処理したり、**動きのある Web ページ**を作るには、**プログラミング言語**が必要なんだ！

Phase 3 では、**JavaScript というプログラミング言語**を使って、Web ページに命を吹き込む方法を学んでいくよ！

### できるようになること

- 変数を使ってデータを保存・操作できる
- 条件分岐で「もし〜なら」の処理が書ける
- ループで同じ処理を繰り返せる
- 関数を使ってコードを整理できる
- 配列とオブジェクトでデータを管理できる
- DOM 操作で HTML を動的に変更できる

**これだけできれば、ボタンをクリックしたら色が変わる、カウントアップする、入力内容を検証する、といった動的な機能が作れるよ！** 💪

## 🚀 Phase 3 の進め方

### 推奨ステップ

1. **Lesson 1 から順番に進める**

   - JavaScript は積み重ねの言語！基礎から順番に学ぶのが大切
   - 焦らず、一つずつ確実に理解していこう
   - わからなくなったら前のレッスンに戻って復習しよう

2. **実際に手を動かす**

   - サンプルコードを自分で書いて動かしてみる
   - 値を変えて、何が起こるか実験する
   - console.log() でたくさん出力して、動作を確認する

3. **ブラウザの開発者ツールを使う**

   - F12 キーで開発者ツールを開く
   - Console タブで JavaScript を直接実行できる
   - エラーメッセージを読んで、何が問題か理解しよう

4. **演習問題に挑戦する**

   - 各レッスンの演習問題を必ず解く
   - AI に相談しながらで OK
   - 解答例と自分のコードを比較して学ぶ

5. **エラーを恐れない**
   - プログラミングではエラーは当たり前！
   - エラーメッセージはあなたの味方（何が間違っているか教えてくれる）
   - エラーを修正する過程で、ぐんぐん成長できる

## 🤖 バイブコーダー向けポイント

### Phase 3 での AI の使い方

Phase 3 では、プログラミングを学ぶけど、AI はあなたの最高のペアプログラマーだよ！

#### 1. コードの生成を依頼する

```
良い指示の例：
「1から10までの数字を順番にconsole.logで出力する
forループを書いてください」

悪い指示の例：
「ループして」
```

#### 2. コードの説明を求める

```
良い質問の例：
「このコードで、iが0から始まって9まで増えていくのはなぜですか？
i < 10という条件の意味を教えてください」

生成されたコードは必ず理解しよう！
```

#### 3. エラーの解決を相談する

```
良い質問の例：
「このコードを実行すると『Uncaught ReferenceError: x is not defined』
というエラーが出ます。原因と修正方法を教えてください」

エラーメッセージをそのまま伝えるのが大事！
```

#### 4. より良い書き方を学ぶ

```
良い質問の例：
「この配列の処理、forループで書いたのですが、
もっと簡潔な書き方はありますか？」

AI から現代的な書き方を学べる！
```

### チェックポイント

AI が生成した JavaScript コードを見る時、こんなポイントをチェックしよう：

- ✅ **変数宣言が適切か**

  - const と let が適切に使い分けられているか
  - var は使われていないか（古い書き方）
  - 変数名が分かりやすいか

- ✅ **ロジックが正しいか**

  - 条件分岐の条件が正しいか
  - ループが無限ループになっていないか
  - 意図した動作をするか console.log で確認

- ✅ **関数が適切に使われているか**

  - 同じコードを繰り返していないか（関数にまとめる）
  - 関数名が何をするか分かりやすいか
  - 引数と戻り値が適切か

- ✅ **エラーハンドリングがあるか**
  - null や undefined のチェックがあるか
  - 配列の範囲外アクセスを防いでいるか
  - ユーザー入力の検証があるか

## 💡 プログラミング学習のコツ

### 1. 小さく始める

最初から複雑なコードを書こうとしない！

```javascript
// まずはシンプルに
console.log("Hello, World!");

// 動いたら少しずつ追加
const name = "太郎";
console.log("Hello, " + name);

// さらに機能を追加
function greet(name) {
  console.log("Hello, " + name);
}
greet("太郎");
```

**小さな成功を積み重ねるのが大事！** 🎉

---

### 2. console.log() を活用する

プログラムの動作を確認する最高の方法：

```javascript
const x = 10;
console.log("xの値:", x); // 変数の中身を確認

const y = x * 2;
console.log("yの値:", y); // 計算結果を確認

// こうやって一歩ずつ確認すると、どこで間違ったかすぐ分かる！
```

**console.log() はプログラマーの必須ツール！** 🔍

---

### 3. エラーメッセージを読む

エラーは敵じゃなくて、先生！

```javascript
// エラーが出たら、メッセージをよく読もう
Uncaught ReferenceError: greeting is not defined
    at script.js:5

// これは「greetingという変数が定義されていません」という意味
// script.jsの5行目で起きてます
```

**エラーメッセージには、問題を解決するヒントが詰まってる！** 💡

---

### 4. 実験する

値を変えたり、コードを改造したりして、何が起こるか試してみよう！

```javascript
// サンプルコード
for (let i = 0; i < 5; i++) {
  console.log(i);
}

// 実験1: 5を10に変えたらどうなる？
// 実験2: i++をi += 2に変えたらどうなる？
// 実験3: console.log(i)をconsole.log(i * 2)に変えたらどうなる？
```

**実験しながら学ぶと、深く理解できる！** 🔬

## 🔍 使用している技術

### HTML（Lesson 1-3 で学んだ内容）

#### ページ構造（すべてのページで共通）

- **基本構造**：DOCTYPE, html, head, body
- **セマンティック要素**：
  - `<header>` - サイトのヘッダー
  - `<nav>` - ナビゲーションメニュー
  - `<main>` - メインコンテンツ
  - `<article>` - 記事（独立したコンテンツ）
  - `<aside>` - サイドバー（補足情報）
  - `<footer>` - サイトのフッター

#### トップページ（index.html）の特徴

```html
<!-- 記事カード（Phase 1 で学んだ article 要素） -->
<article class="article-card">
  <h3><a href="article1.html">記事タイトル</a></h3>
  <p class="article-meta">2025年1月15日 投稿</p>
  <p class="article-excerpt">記事の概要...</p>
  <a href="article1.html" class="read-more">続きを読む →</a>
</article>
```

#### 記事詳細ページの特徴

```html
<!-- セマンティックな記事構造 -->
<article class="article-detail">
  <header class="article-header">
    <!-- 記事のタイトルとメタ情報 -->
  </header>
  <section class="article-content">
    <!-- 記事本文 -->
  </section>
  <section class="related-articles">
    <!-- 関連記事リンク -->
  </section>
</article>
```

### CSS（Lesson 4-5 で学んだ内容）

#### 1. セレクタ

```css
/* 要素セレクタ（Phase 1 で学んだ） */
header {
  background-color: #1f2937;
}

/* クラスセレクタ（Phase 1 で学んだ） */
.article-card {
  padding: 1.5rem;
}

/* 疑似クラス（Phase 1 で学んだ） */
a:hover {
  text-decoration: underline;
}
```

#### 2. ボックスモデル

```css
/* Phase 1 で学んだボックスモデル */
.article-card {
  padding: 1.5rem; /* 内側の余白 */
  margin-bottom: 1.5rem; /* 外側の余白 */
  border: 1px solid #e5e7eb; /* 枠線 */
  border-radius: 8px; /* 角を丸くする */
}
```

#### 3. テキストスタイリング

```css
/* Phase 1 で学んだテキストスタイリング */
header h1 {
  font-size: 2.5rem; /* rem 単位 */
  font-weight: bold; /* 太字 */
  color: white; /* 文字色 */
}

.article-excerpt {
  line-height: 1.6; /* 行間 */
  color: #6b7280; /* 16 進数カラー */
}
```

#### 4. リンクのスタイリング

```css
/* Phase 1 で学んだリンクのスタイリング */
a {
  color: #3b82f6; /* リンクの色 */
  text-decoration: none; /* 下線を消す */
}

a:hover {
  color: #3b82f6; /* ホバー時の色 */
  text-decoration: underline; /* ホバー時に下線を表示 */
}
```

#### 5. レイアウト

```css
/* Phase 1 で学んだレイアウト */
.container {
  max-width: 1000px;
  margin: 0 auto; /* 中央寄せ */
}

.nav-list li {
  display: inline-block; /* 横並び */
  margin-right: 2rem;
}
```

## 🌟 このプロジェクトの特徴

### 1. 複数ページのサイト構造

トップページ（index.html）から個別記事ページ（article1-3.html）へのリンクがあり、実際のブログのような構造になっています。

**学べること**：

- 複数の HTML ファイル間のナビゲーション
- 一貫性のあるデザインを保つ方法（外部 CSS の活用）
- 相対パス（`href="article1.html"`）の使い方

### 2. セマンティック HTML の実践

すべてのページで `<header>`, `<nav>`, `<main>`, `<article>`, `<aside>`, `<footer>` などのセマンティックな要素を使っています。

**学べること**：

- ページ構造の意味を明確にする方法
- SEO とアクセシビリティに優れたコードの書き方
- `<div>` に頼らない、意味のあるマークアップ

### 3. 統一感のあるデザイン

1 つの CSS ファイル（styles.css）ですべてのページのスタイルを管理しています。

**学べること**：

- 外部 CSS の効率的な使い方
- 共通のスタイルを定義して再利用する方法
- 一貫性のあるデザインを保つ方法

### 4. 実践的な要素

記事一覧、記事詳細、関連記事リンク、ナビゲーションメニューなど、実際のブログで使われる要素を実装しています。

**学べること**：

- 実際のサイトで使われるデザインパターン
- ユーザビリティを考慮した構造
- 読みやすい記事レイアウト

## 💡 学習のポイント

### 1. 複数ページでの一貫性

このプロジェクトでは、4 つの HTML ファイルが同じ CSS ファイル（styles.css）を読み込んでいます。

**メリット**：

- CSS を 1 箇所変更するだけで、全ページに反映される
- デザインの一貫性が保たれる
- コードの重複が減る

**例**：

```css
/* styles.css で header のスタイルを変更すると、
   すべてのページのヘッダーが変わる */
header {
  background-color: #3b82f6; /* 青に変更 */
}
```

### 2. セマンティック HTML の重要性

このプロジェクトでは、`<div>` ではなくセマンティックな要素を使っています。

**比較**：

```html
<!-- 悪い例 -->
<div class="header">
  <div class="nav">...</div>
</div>

<!-- 良い例 -->
<header>
  <nav>...</nav>
</header>
```

**メリット**：

- コードが読みやすい
- SEO（検索エンジン最適化）に有利
- スクリーンリーダーに優しい

### 3. リンクの設計

このプロジェクトでは、以下のようなリンクがあります：

```html
<!-- トップページから記事へ -->
<a href="article1.html">HTMLの基礎を学ぼう</a>

<!-- 記事からトップページへ -->
<a href="index.html">← 記事一覧に戻る</a>

<!-- 記事から記事へ（関連記事） -->
<a href="article2.html">CSSでデザインしよう</a>
```

**ポイント**：

- 相対パスを使って、同じディレクトリ内のファイルを参照
- `←` や `→` で視覚的に分かりやすくする
- リンクテキストは明確に（「こちら」ではなく、「記事一覧に戻る」）

## 🤖 AI との協働

このプロジェクトを AI と一緒にカスタマイズする時の指示例：

### 良い指示の例

```
「WebDev Blogに『JavaScript入門』という新しい記事を追加してください。
article4.htmlとして作成し、以下の内容を含めてください：
- JavaScriptとは何か
- 基本的な文法（変数、条件分岐、ループ）
- HTMLとの連携方法

スタイルは既存の記事（article1.html, article2.html）と同じにして、
トップページ（index.html）の記事一覧にも追加してください。」
```

**ポイント**：

- 新しいファイル名を明示（article4.html）
- 含めるべき内容を箇条書き
- 既存のファイルとの統一性を指示
- どのファイルを更新すべきか明示（index.html）

### 生成されたコードのチェックポイント

1. **HTML 構造が既存の記事と統一されているか**

   - `<header>`, `<nav>`, `<main>`, `<article>`, `<footer>` が使われているか
   - class 名が既存のものと同じか（article-detail, article-header など）

2. **CSS ファイルが正しくリンクされているか**

   ```html
   <link rel="stylesheet" href="styles.css" />
   ```

3. **リンクが正しく設定されているか**

   - トップページへのリンク（`href="index.html"`）
   - 他の記事へのリンク（関連記事）

4. **トップページ（index.html）に新しい記事が追加されているか**
   - 記事カードが追加されているか
   - リンクが正しいか（`href="article4.html"`）

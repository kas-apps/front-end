セマンティックHTMLの重要性 - なぜ意味のあるタグを使うべきか

2025年1月5日 投稿

セマンティックHTMLとは？

セマンティック（semantic）とは「意味のある」という意味です。セマンティックHTMLとは、 タグの見た目ではなく、「意味」に基づいて適切な要素を使うことを指します。

なぜセマンティックHTMLが重要なのか？

1. 検索エンジンに優しい（SEO向上）

Googleなどの検索エンジンは、セマンティックなタグを理解して、ページの内容を正しく評価します。 例えば、<header>や<article>を使うことで、「ここがヘッダー」「ここが記事本文」と伝わります。

2. アクセシビリティの向上

視覚障害のある人が使うスクリーンリーダーは、セマンティックなタグを読み取って、 ページの構造を音声で伝えます。<nav>を使えば「ここはナビゲーション」と分かります。

3. コードの可読性が上がる

<div>だらけのコードよりも、<header>、<main>、<footer>を使った方が、 他の開発者（未来の自分も！）がコードを理解しやすくなります。

主要なセマンティック要素

要素 意味 用途
<header> ヘッダー ページやセクションの導入部
<nav> ナビゲーション メニュー、リンク集
<main> メインコンテンツ ページの主要な内容
<article> 記事 独立したコンテンツ
<section> セクション 内容のまとまり
<aside> サイドバー 補足情報、関連リンク
<footer> フッター ページやセクションの末尾

悪い例と良い例

悪い例（divだらけ）

  • <div class="header"> - 意味が分からない
  • <div class="nav"> - CSSでしか分からない
  • <div class="content"> - 曖昧

良い例（セマンティック）

  • <header> - ヘッダーだと一目で分かる
  • <nav> - ナビゲーションだと明確
  • <main> - メインコンテンツだと分かる

まとめ

セマンティックHTMLを使うことで、検索エンジン、スクリーンリーダー、他の開発者に優しいWebページを作れます。 見た目は同じでも、「意味」を持たせることで、より質の高いコードになります。

## 🔍 使用している技術

### HTML（Lesson 1-3 で学んだ内容）

- **基本構造**：DOCTYPE, html, head, body
- **セマンティック要素**：header, main, section, footer
- **テキスト要素**：h1, h2, h3, p
- **リスト**：ul, li
- **テーブル**：table, thead, tbody, tr, th, td
- **リンク**：a タグ（mailto, target="\_blank"）
- **コンテナ**：div（hobby-list, hobby-item）

### CSS（Lesson 4-5 で学んだ内容）

#### 1. セレクタ

- **要素セレクタ**：`body`, `header`, `section`, `p` など
- **クラスセレクタ**：`.subtitle`, `.hobby-list`, `.contact-list` など
- **疑似クラス**：`:hover`（リンクのホバー効果）

#### 2. ボックスモデル

```css
/* Lesson 4 で学んだボックスモデル */
section {
  padding: 2rem; /* 内側の余白 */
  margin-bottom: 2rem; /* 外側の余白 */
  border-radius: 8px; /* 角を丸くする */
}
```

#### 3. テキストスタイリング

```css
/* Lesson 5 で学んだテキストスタイリング */
header h1 {
  font-size: 2.5rem; /* rem 単位 */
  font-weight: bold; /* 太字 */
  letter-spacing: 0.05em; /* 文字間隔 */
  color: white; /* 文字色 */
}

section p {
  line-height: 1.6; /* 行間 */
  color: #6b7280; /* 16 進数カラー */
}
```

#### 4. リンクのスタイリング

```css
/* Lesson 5 で学んだリンクのスタイリング */
a {
  color: #3b82f6; /* リンクの色 */
  text-decoration: none; /* 下線を消す */
}

a:hover {
  text-decoration: underline; /* ホバー時に下線を表示 */
}
```

#### 5. レイアウト

```css
/* Lesson 4 で学んだ display プロパティ */
main {
  max-width: 800px;
  margin: 2rem auto; /* 中央寄せ */
}
```

## 💡 学習のポイント

### 1. セマンティック HTML の重要性

このプロジェクトでは、`header`, `main`, `section`, `footer` などのセマンティックな要素を使っています。

**メリット**：

- コードが読みやすい
- SEO（検索エンジン最適化）に有利
- アクセシビリティが向上

**悪い例**：

```html
<div class="header">
  <div class="title">山田太郎</div>
</div>
```

**良い例**：

```html
<header>
  <h1>山田太郎</h1>
</header>
```

### 2. CSS の再利用性

同じスタイルを複数の要素に適用する時は、**クラスセレクタ**を使いましょう。

```css
/* すべての section に共通のスタイル */
section {
  background-color: white;
  padding: 2rem;
  margin-bottom: 2rem;
}
```

### 3. rem 単位の活用

このプロジェクトでは、フォントサイズに `rem` 単位を使っています。

**メリット**：

- ルート要素（html）の font-size を変えるだけで、全体のサイズを一括変更できる
- アクセシビリティに優れている（ユーザーのフォント設定を尊重）

```css
html {
  font-size: 16px; /* これを 18px に変えると、全体が大きくなる */
}

h1 {
  font-size: 2.5rem; /* 16px × 2.5 = 40px */
}
```

## 🤖 AI との協働

このプロジェクトを AI と一緒にカスタマイズする時の指示例：

### 良い指示の例

```
「プロフィールページに『ブログ』セクションを追加してください。
最新の3つの記事をリスト形式で表示し、各記事にタイトル、公開日、
概要を含めてください。スタイルは既存の.hobby-itemクラスと
同じような見た目にしてください。」
```

**ポイント**：

- 追加する内容を具体的に指定
- スタイルの参考（既存のクラス）を明示
- 表示する情報を列挙

### 生成されたコードのチェックポイント

1. **HTML 構造が正しいか**

   - セマンティックな要素が使われているか
   - タグが正しく閉じられているか

2. **CSS が既存のスタイルと統一されているか**

   - 色、フォントサイズ、余白が既存のセクションと合っているか
   - クラス名が分かりやすいか

3. **コメントが適切に付いているか**
   - 新しく追加された部分にコメントがあるか

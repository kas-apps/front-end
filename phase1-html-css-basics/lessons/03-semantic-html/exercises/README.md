## 基礎編

### 演習 1：簡単な記事ページを作ろう

以下の構造で HTML を作成してください：

- header：サイト名（h1）
- main：
  - article：記事タイトル（h2）、本文（p）
- footer：著作権表示

**ヒント**：まず header, main, footer の大枠を作ってから、中身を埋めよう！

**解答例**：[03-01.html](solutions/03-01.html)

### 演習 2：ブログトップページを作ろう

以下の要素を含むページを作成してください：

- header：サイト名、nav（3 つのリンク）
- main：3 つの article（各 article に h2, p）
- aside：「人気の記事」見出しと 3 つのリンク
- footer：著作権表示

**解答例**：[03-02.html](solutions/03-02.html)

### 演習 3：AI に指示して作ってもらおう

AI に以下の内容のページを作ってもらってください：

```text
「レシピページを作ってください。
articleタグで全体を囲み、
headerタグに料理名（h1）と調理時間、
sectionタグで『材料』『作り方』の2セクション、
それぞれh2見出しとulリストを含めてください。」
```

生成されたコードを、レッスンで学んだチェックリストで確認してみよう！

#### ✅ チェックリスト

1. **適切なセマンティック要素が使われているか**
   - header, article, section などが正しく使われている
2. **見出しの階層が正しいか**
   - h1 → h2 の順になっている
3. **レシピとして論理的な構造か**
   - 材料と作り方が明確に分かれている

**解答例**：[03-03.html](solutions/03-03.html)

## 応用編

### 演習 4：正しい要素を選ぼう

以下のコンテンツに、最適なセマンティック要素を選んでください：

1. ニュース記事 → `<?>` ? `</?>` ?
2. サイトのメインメニュー → `<?>` ? `</?>` ?
3. ページの主要コンテンツ → `<?>` ? `</?>` ?
4. 記事の補足情報（コラム） → `<?>` ? `</?>` ?
5. 記事内の各章 → `<?>` ? `</?>` ?

<details>
<summary>答えを見る</summary>

1. `<article>` - 単独で意味が完結するから
2. `<nav>` - ナビゲーションリンクだから
3. `<main>` - ページの主要コンテンツだから
4. `<aside>` - 補足情報だから
5. `<section>` - 大きな文書の一部だから

</details>

### 演習 5：エラーを見つけて修正しよう

以下の HTML には問題があります。何が間違っているか見つけて修正してください：

```html
<body>
  <div class="header">
    <h1>私のサイト</h1>
  </div>

  <main>
    <article>
      <h2>記事1</h2>
      <p>内容...</p>
    </article>
  </main>

  <main>
    <article>
      <h2>記事2</h2>
      <p>内容...</p>
    </article>
  </main>

  <div class="sidebar">
    <h3>カテゴリー</h3>
    <ul>
      <li>技術</li>
      <li>日記</li>
    </ul>
  </div>

  <div class="footer">
    <p>&copy; 2025</p>
  </div>
</body>
```

**ヒント**：少なくとも 4 つの問題があるよ！

<details>
<summary>答えを見る</summary>

**問題点**：

1. `<div class="header">` → `<header>` にすべき
2. `<main>` が 2 つある → 1 つにまとめる
3. `<div class="sidebar">` → `<aside>` にすべき
4. `<div class="footer">` → `<footer>` にすべき

**修正版**は[こちら](solutions/03-05.html)を参照

</details>

## チャレンジ

### 演習 6：ポートフォリオサイトの構造を設計しよう

あなたのポートフォリオサイトの HTML 構造を設計してください。

**含める要素**：

- サイト全体のヘッダー（名前、ナビゲーション）
- 自己紹介セクション
- スキル・経験のセクション
- プロジェクト紹介（複数の記事）
- お問い合わせセクション
- フッター

**条件**：

- 適切なセマンティック要素を使う
- 論理的な階層構造にする
- 見出しレベル（h1-h6）を正しく使う

完成したら、AI に「この構造で実際の HTML を生成して」と指示してみよう！

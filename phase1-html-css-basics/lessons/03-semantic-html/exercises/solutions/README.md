#### [03-01.html](03-01.html) - 演習 1 の解答：簡単な記事ページ

**演習内容**：

- header：サイト名（h1）
- main：article：記事タイトル（h2）、本文（p）
- footer：著作権表示

**ポイント**：

- ✅ ページ全体の大枠を header, main, footer で構造化
- ✅ 記事を article タグで囲む（単独で意味が完結するため）
- ✅ main タグはページに 1 つだけ
- ✅ コメントで各セクションの役割を明示

**セマンティック要素の使い分け**：

- `<header>` - サイト全体のヘッダー
- `<main>` - ページの主要コンテンツ
- `<article>` - 単独で意味が完結するコンテンツ
- `<footer>` - サイト全体のフッター

#### [03-02.html](03-02.html) - 演習 2 の解答：ブログトップページ

**演習内容**：

- header：サイト名、nav（3 つのリンク）
- main：3 つの article（各 article に h2, p）
- aside：「人気の記事」見出しと 3 つのリンク
- footer：著作権表示

**ポイント**：

- ✅ nav タグでナビゲーションメニューを明示
- ✅ main 内に複数の article を配置
- ✅ aside で補足情報（サイドバー）を表現
- ✅ 各 article は独立した記事として機能

**セマンティック要素の使い分け**：

- `<nav>` - ナビゲーションリンクの集まり
- `<article>` - 複数の記事を並べる（各記事が独立している）
- `<aside>` - メインコンテンツとは別の補足情報

**よくある間違い**：

- ❌ aside を main の中に入れてしまう → aside は main と並列に配置
- ❌ nav を ul だけで実装 → nav タグで囲むことで意味が明確に

**演習内容**：

- `<header>` - レシピのタイトル部分（article 内のヘッダー）
- `<section>` - レシピ内の各セクション（材料、作り方）

**AI への指示のコツ**：

- セマンティック要素名を具体的に指定する

## 応用編

### 演習 4：正しい要素を選ぶ - 解説

**解答**：

1. ニュース記事 → `<article>`
   - 理由：単独で意味が完結し、RSS フィードなどで配信できる
2. サイトのメインメニュー → `<nav>`
   - 理由：ナビゲーションリンクの集まりだから
3. ページの主要コンテンツ → `<main>`
   - 理由：ページに 1 つだけ、主要なコンテンツを示す
4. 記事の補足情報（コラム） → `<aside>`
   - 理由：メインコンテンツとは独立した補足情報
5. 記事内の各章 → `<section>`
   - 理由：大きな文書の一部を論理的にグループ化

**判断基準**：

- **単独で完結する？** → article
- **ナビゲーション？** → nav
- **ページの主要部分？** → main
- **補足情報？** → aside
- **論理的なグループ？** → section

#### [03-05.html](03-05.html) - 演習 5 の解答：エラー修正版

**問題点と修正**：

1. **`<div class="header">` → `<header>`**

   - div に class で意味を付けるのではなく、セマンティック要素を使う
   - ブラウザやスクリーンリーダーが正しく理解できる

2. **`<main>` が 2 つある**

   - main はページに 1 つだけ
   - 複数の article は 1 つの main の中に配置

3. **`<div class="sidebar">` → `<aside>`**

   - サイドバーのような補足情報は aside を使う

4. **`<div class="footer">` → `<footer>`**
   - フッターには footer タグを使う

**重要な教訓**：

- div に class で意味を付けるより、セマンティック要素を使う
- main タグはページに 1 つだけ
- 「意味があるもの」にはセマンティック要素を使おう

## チャレンジ

#### [03-06.html](03-06.html) - 演習 6 の解答：ポートフォリオサイト

**演習内容**：

ポートフォリオサイトの HTML 構造を設計

**ポイント**：

- ✅ サイト全体を header, main, footer で構造化
- ✅ nav でナビゲーションメニュー（ページ内リンク）
- ✅ main 内に複数の section で各セクションを分ける
- ✅ スキルと経験は article でグループ化
- ✅ 各プロジェクトも article として独立させる
- ✅ 見出しレベル（h1 → h2 → h3）を正しく使用

**セマンティック要素の使い分け**：

- `<header>` + `<nav>` - サイトヘッダーとメニュー
- `<main>` - ページの主要コンテンツ全体
- `<section>` - About, Skills, Projects, Contact などの各セクション
- `<article>` - 独立したコンテンツ（スキル、経験、各プロジェクト）
- `<footer>` - 著作権情報

**構造設計のポイント**：

1. **大枠から考える**

   - header, main, footer の大きな構造
   - main 内をどうセクション分けするか

2. **section と article の使い分け**

   - section: ページの論理的な区切り（About, Skills など）
   - article: 単独で意味が完結するもの（各プロジェクトなど）

3. **見出しの階層を守る**

   - h1: ページのメインタイトル（名前）
   - h2: 各セクションのタイトル（About, Skills など）
   - h3: セクション内の小見出し（各プロジェクト名など）

4. **ナビゲーションの設計**
   - nav タグで囲む
   - ページ内リンク（#about, #skills など）を活用

## 🎯 学習のポイント

これらの解答例を見る時は、以下のポイントに注目しよう：

### 1. セマンティック要素の選び方

**判断フローチャート**：

```
コンテンツの種類は？
├─ ナビゲーションリンク → <nav>
├─ ページの主要部分 → <main>
├─ 単独で完結する記事 → <article>
├─ 論理的なセクション → <section>
├─ 補足情報 → <aside>
├─ ヘッダー部分 → <header>
├─ フッター部分 → <footer>
└─ 特に意味がない → <div> か <span>
```

### 2. div との使い分け

**セマンティック要素を使うべき場合**：

- 明確な役割がある（header, nav, main, article, section, aside, footer）
- 検索エンジンやスクリーンリーダーに理解させたい

**div を使ってもいい場合**：

- 純粋にスタイリングのためのグループ化
- 特に意味的な役割がない場合

### 3. よくある間違いとその修正

| 間違い                      | 問題点                     | 修正方法                                    |
| --------------------------- | -------------------------- | ------------------------------------------- |
| `<div class="header">`      | 意味が伝わらない           | `<header>` を使う                           |
| `<main>` が複数ある         | main はページに 1 つだけ   | 1 つの main にまとめる                      |
| article と section の混同   | 使い分けが分かっていない   | 単独で完結するか？ で判断                   |
| nav を使わずに ul だけ      | ナビゲーションが不明確     | `<nav>` で囲む                              |
| aside を main の中に入れる  | aside は main と並列に配置 | aside を main の外に出す                    |
| header, footer を divで代用 | セマンティックではない     | 適切な要素（header, footer）を使う          |
| 見出しレベルを飛ばす        | 階層が不明確               | h1 → h2 → h3 の順に使う（h4 以降も飛ばさない） |

### 4. 構造設計の手順

1. **大枠を決める**

   ```html
   <header></header>
   <main></main>
   <footer></footer>
   ```

2. **main の中を分ける**

   ```html
   <main>
     <section></section>
     <section></section>
   </main>
   ```

3. **各セクションの中身を作る**

   ```html
   <section>
     <h2>タイトル</h2>
     <article>...</article>
   </section>
   ```

4. **見出しレベルを整理**
   - h1: ページ全体のタイトル
   - h2: 各セクションのタイトル
   - h3: セクション内の小見出し

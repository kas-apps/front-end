# Lesson 3: セマンティック HTML 🏗️

**学習目標**：HTML の「意味」を理解し、適切なセマンティック要素を使ってドキュメント構造を作れるようになる

---

## セマンティック要素の基礎知識

### セマンティック（Semantic）って何？

**Semantic = 意味のある**

セマンティック要素とは、**「ただの箱」じゃなくて「何の箱か」が分かるタグ**のこと。

```html
<!-- divは意味が分からない -->
<div>コンテンツ</div>

<!-- headerは意味が分かる！ -->
<header>コンテンツ</header>
```

**例え話**：

- `<div>`：透明な収納ボックス（中身が何か分からない）
- `<header>`：「書類」って書いてある収納ボックス（一目で分かる！）

---

### 重要：ブラウザでの見た目は同じ！

ここが超重要なポイント！👀

**セマンティック要素を使っても、ブラウザでの表示は`<div>`と全く同じ**なんだ！

```html
<!-- どちらもブラウザでの表示は同じ -->
<div>これはdiv</div>
<header>これはheader</header>
<main>これはmain</main>
<footer>これはfooter</footer>
```

**全部、ただの四角い箱として表示される。**

色も付かないし、サイズも変わらない。見た目は完全に同じ。

### じゃあ、なぜセマンティック要素を使うの？

**「見た目が同じなら、div でいいじゃん！」** って思うよね？

でも実は、**見た目以外のところで大きな違いがある**んだ！

次のセクションで、その理由を詳しく見ていこう！ 🚀

---

## なぜセマンティック HTML を学ぶの？

さっき「見た目は同じ」って言ったけど、セマンティック要素には**見た目以外の大きなメリット**があるんだ！

### セマンティック HTML を使う 4 つのメリット

#### 1. 検索エンジンが内容を理解できる 🔍

```html
<!-- 悪い例：divだけ -->
<div class="header">
  <div class="title">私のブログ</div>
</div>
<div class="content">
  <div class="post">
    <div class="post-title">記事タイトル</div>
    <div class="post-body">記事本文...</div>
  </div>
</div>

<!-- 良い例：セマンティック要素 -->
<header>
  <h1>私のブログ</h1>
</header>
<main>
  <article>
    <h2>記事タイトル</h2>
    <p>記事本文...</p>
  </article>
</main>
```

**Google にはどう見える？**

- 悪い例：「何かの箱がいっぱいあるな...」
- 良い例：「ヘッダーがあって、メインコンテンツに記事がある！ちゃんと構造化されてるね！」

#### 2. アクセシビリティが向上する ♿

視覚障害のある人が使うスクリーンリーダーは、セマンティック要素を頼りにページを理解するよ。

```html
<nav>
  <ul>
    <li><a href="/">ホーム</a></li>
    <li><a href="/about">概要</a></li>
  </ul>
</nav>
```

スクリーンリーダー：「ナビゲーションエリアに入りました」
→ ユーザーは「あ、ここにメニューがあるんだな」と分かる！

#### 3. AI への指示が的確になる 🤖

```text
❌ 曖昧な指示：
「ページの上の方に、タイトルとメニューを作って」

⭕ 的確な指示：
「headerタグの中にh1でサイトタイトル、
navタグでメニューを作ってください」
```

**結果**：一発で、適切な構造の HTML が生成される！

#### 4. コードが読みやすくなる 📖

チームで開発する時、半年後の自分が見る時、どっちが分かりやすい？

```html
<!-- 分かりにくい -->
<div class="header">...</div>
<div class="main">...</div>
<div class="footer">...</div>

<!-- 一目瞭然！ -->
<header>...</header>
<main>...</main>
<footer>...</footer>
```

**まるで、コードが自分で説明してくれてるみたい！** ✨

## 主要なセマンティック要素

### 1. `<header>` - ヘッダー

```html
<header>
  <h1>私のウェブサイト</h1>
  <p>素敵なサイトへようこそ</p>
</header>
```

- **役割**：ページやセクションの導入部分
- **含むもの**：サイトタイトル、ロゴ、キャッチフレーズなど
- **位置**：ページの上部、または記事の上部

**⚠️ 注意：`<head>`と混同しやすい！**

初めて見る人がよく間違えるのが、`<head>`と`<header>`の違い。

- **`<head>`**：ページの設定情報（Lesson 1 で学んだやつ！）
  - **表示されない**
  - 文字コード、タイトル、CSS リンクなど
- **`<header>`**：ページの導入部分
  - **表示される**
  - サイトロゴ、タイトル、ナビゲーションなど

**例え話**：本でいうと、`<head>`は奥付（本のデータ）、`<header>`は表紙・前書き！

---

### 2. `<nav>` - ナビゲーション

```html
<nav>
  <ul>
    <li><a href="/">ホーム</a></li>
    <li><a href="/about">私について</a></li>
    <li><a href="/blog">ブログ</a></li>
    <li><a href="/contact">お問い合わせ</a></li>
  </ul>
</nav>
```

- **役割**：サイト内の主要なリンク集
- **含むもの**：メインメニュー、パンくずリスト
- **位置**：ヘッダー内、サイドバー、フッターなど

**注意**：ページ内のすべてのリンクを`<nav>`で囲む必要はない！

- メインメニュー：`<nav>` ⭕
- 記事内のリンク：`<nav>` ❌（普通に`<a>`で OK）

---

### 3. `<main>` - メインコンテンツ

```html
<main>
  <h1>ページのメインタイトル</h1>
  <p>ここがページの主要な内容です。</p>
</main>
```

- **役割**：ページの主要コンテンツ
- **ルール**：**1 ページに 1 つだけ**！
- **含まないもの**：ヘッダー、フッター、サイドバーなど（繰り返し使う要素）

**例え話**：本の「本文」みたいなもの。表紙やページ番号は含まない。

---

### 4. `<article>` - 独立したコンテンツ

```html
<article>
  <h2>ブログ記事のタイトル</h2>
  <p>投稿日：2025年1月15日</p>
  <p>記事の本文がここに入ります...</p>
</article>
```

- **役割**：単体で意味が完結するコンテンツ
- **使う場面**：ブログ記事、ニュース記事、商品レビュー、コメント
- **判断基準**：「これだけを別のサイトに持っていっても意味が通じる？」→ YES なら`<article>`

**例**：

- ブログの 1 つの記事 → `<article>` ⭕
- サイト全体の説明 → `<article>` ❌（`<section>`か`<div>`）

---

### 5. `<section>` - セクション

```html
<section>
  <h2>私のスキル</h2>
  <p>HTML、CSS、JavaScriptが得意です。</p>
</section>

<section>
  <h2>趣味</h2>
  <p>読書と映画鑑賞が好きです。</p>
</section>
```

- **役割**：テーマごとのまとまり
- **必須**：原則として**見出し（h2-h6）を含む**
- **使う場面**：章立て、トピックごとの区切り

**`<article>`との違い**：

- `<article>`：単独で完結する（記事、投稿）
- `<section>`：大きな文書の一部（章、セクション）

---

### 6. `<aside>` - 補足情報

```html
<aside>
  <h3>関連記事</h3>
  <ul>
    <li><a href="/post1">関連記事1</a></li>
    <li><a href="/post2">関連記事2</a></li>
  </ul>
</aside>
```

- **役割**：メインコンテンツに関連する補足情報
- **使う場面**：サイドバー、関連リンク、用語解説、広告
- **特徴**：なくてもメインコンテンツは理解できる

**例え話**：本の「コラム」や「補足説明」みたいなもの。

---

### 7. `<footer>` - フッター

```html
<footer>
  <p>&copy; 2025 私のウェブサイト. All rights reserved.</p>
  <nav>
    <a href="/privacy">プライバシーポリシー</a>
    <a href="/terms">利用規約</a>
  </nav>
</footer>
```

- **役割**：ページやセクションの終わり・補足情報
- **含むもの**：著作権表示、連絡先、関連リンク
- **位置**：ページの最下部、または記事の下部

## コンテナ要素（div, span）との違い

Lesson 2 で学んだ`<div>`と`<span>`を覚えてる？

ここまでセマンティック要素のメリットを説明してきたけど、「じゃあ div と span はもう使わないの？」って思うかもしれないね。

### `<div>` と `<span>` は悪じゃない！

**答えは NO！** `<div>`と`<span>`も引き続き大事な役割があるんだ。

ここでは、**セマンティック要素とコンテナ要素の使い分け**を学ぼう！

### `<div>` - ブロックレベルのコンテナ

```html
<!-- 良い使い方：CSSでスタイリングするための箱 -->
<div class="card">
  <h3>カードタイトル</h3>
  <p>カードの内容</p>
</div>
```

**使う場面**：

- スタイリングのためのグループ化
- レイアウト用の箱
- **他に適切なセマンティック要素がない時**

### `<span>` - インラインレベルのコンテナ

```html
<p>この部分は<span class="highlight">重要</span>です。</p>
```

**使う場面**：

- テキストの一部にスタイルを適用
- インラインでグループ化が必要な時

### 使い分けの基準

```text
質問1：「これは何？」という意味がある？
  YES → セマンティック要素を使う（header, article など）
  NO  → 次の質問へ

質問2：スタイリングやレイアウトのため？
  YES → div または span でOK
  NO  → そもそも要素が不要かも
```

**良い例**：

```html
<!-- 意味がある部分はセマンティック要素 -->
<header>
  <h1>サイトタイトル</h1>
</header>

<!-- レイアウト用の箱はdiv -->
<div class="container">
  <div class="row">
    <!-- コンテンツ -->
  </div>
</div>

<!-- 記事はarticle -->
<article>
  <h2>記事タイトル</h2>
  <p>本文...</p>
</article>
```

## 実践的な HTML 構造の例

### 完全なページ構造

```html
<!DOCTYPE html>
<html lang="ja">
  <head>
    <meta charset="UTF-8" />
    <title>私のブログ</title>
  </head>
  <body>
    <!-- ページ全体のヘッダー -->
    <header>
      <h1>私のブログ</h1>
      <p>日々の学びを記録しています</p>

      <!-- メインナビゲーション -->
      <nav>
        <ul>
          <li><a href="/">ホーム</a></li>
          <li><a href="/about">私について</a></li>
          <li><a href="/contact">お問い合わせ</a></li>
        </ul>
      </nav>
    </header>

    <!-- メインコンテンツ（1ページに1つ） -->
    <main>
      <!-- 記事1 -->
      <article>
        <header>
          <h2>HTML を学んで1週間</h2>
          <p>投稿日：2025年1月15日</p>
        </header>

        <section>
          <h3>学んだこと</h3>
          <p>セマンティック HTML の重要性を学びました。</p>
        </section>

        <section>
          <h3>次の目標</h3>
          <p>CSS のレイアウトを習得します。</p>
        </section>

        <footer>
          <p>カテゴリ：学習記録</p>
        </footer>
      </article>

      <!-- 記事2 -->
      <article>
        <header>
          <h2>初めてのウェブサイト公開</h2>
          <p>投稿日：2025年1月20日</p>
        </header>

        <p>ついに自分のサイトを公開しました！</p>

        <footer>
          <p>カテゴリ：達成記録</p>
        </footer>
      </article>
    </main>

    <!-- サイドバー（補足情報） -->
    <aside>
      <h3>人気の記事</h3>
      <ul>
        <li><a href="/post1">記事1</a></li>
        <li><a href="/post2">記事2</a></li>
      </ul>
    </aside>

    <!-- ページ全体のフッター -->
    <footer>
      <p>&copy; 2025 私のブログ</p>
      <nav>
        <a href="/privacy">プライバシーポリシー</a>
        <a href="/terms">利用規約</a>
      </nav>
    </footer>
  </body>
</html>
```

### 構造の可視化

```text
body
├── header (ページヘッダー)
│   ├── h1 (サイトタイトル)
│   └── nav (メインメニュー)
│
├── main (メインコンテンツ)
│   ├── article (記事1)
│   │   ├── header (記事ヘッダー)
│   │   ├── section (セクション1)
│   │   ├── section (セクション2)
│   │   └── footer (記事フッター)
│   │
│   └── article (記事2)
│       ├── header
│       └── footer
│
├── aside (サイドバー)
│   └── 関連情報
│
└── footer (ページフッター)
    └── 著作権・リンク
```

**まるでビルの設計図みたい！** 各階（要素）の役割が明確だね！🏗️

## 🤖 バイブコーディング実践

### AI への指示例

セマンティック HTML を AI に生成してもらう時の指示例だよ！

#### ⭕ 良い指示の例

```text
「ブログのトップページを作ってください。
- headerタグにサイトタイトル（h1）とnavでメニュー（ホーム、ブログ、お問い合わせ）
- mainタグの中に3つのarticleタグで記事プレビュー
  各articleには見出し（h2）、投稿日、要約（p）を含める
- asideタグにカテゴリー一覧
- footerタグに著作権表示
言語は日本語でお願いします。」
```

**ポイント**：

- どのセマンティック要素を使うか明示
- 各要素に何を含めるか具体的に
- 構造が明確

#### ❌ 曖昧な指示の例

```text
「ブログっぽいページを作って」
```

**問題点**：

- どんな構造か不明
- div だらけのコードになる可能性
- セマンティック要素が使われない

#### 💡 応用的な指示

```text
「記事詳細ページを作ってください。
articleタグで記事全体を囲み、
その中にheaderタグで記事タイトル（h1）と投稿日、
本文は複数のsectionタグで章立て（各sectionにh2見出し）、
最後にfooterタグで著者情報とタグ一覧を表示してください。」
```

**すごい！** セマンティックな構造を明確に指示してる！

### 生成されたコードの読み方

AI がコードを生成したら、以下をチェックしよう！

#### ✅ チェックリスト

1. **適切なセマンティック要素が使われているか？**

   ```html
   <!-- 良い例 -->
   <header>
     <h1>サイト名</h1>
   </header>

   <!-- 悪い例 -->
   <div class="header">
     <h1>サイト名</h1>
   </div>
   ```

   → div の代わりに header が使われてる？ ✅

2. **main タグは 1 つだけか？**

   ```html
   <!-- 悪い例：mainが複数 -->
   <main>...</main>
   <main>...</main>
   <!-- これはNG！ -->
   ```

3. **article と section が正しく使い分けられているか？**

   ```html
   <!-- 良い例：ブログ記事 -->
   <article>
     <h2>記事タイトル</h2>
     <section>
       <h3>導入</h3>
       <p>...</p>
     </section>
     <section>
       <h3>本論</h3>
       <p>...</p>
     </section>
   </article>
   ```

4. **nav はメインナビゲーションに使われているか？**

   ```html
   <!-- 良い例 -->
   <nav>
     <ul>
       <li><a href="/">ホーム</a></li>
       <li><a href="/about">概要</a></li>
     </ul>
   </nav>

   <!-- 悪い例：ただのリンク集をnavで囲んでる -->
   <nav>
     <a href="/random1">リンク1</a>
     <a href="/random2">リンク2</a>
   </nav>
   ```

5. **section には見出しがあるか？**

   ```html
   <!-- 良い例 -->
   <section>
     <h2>セクションのタイトル</h2>
     <p>内容...</p>
   </section>

   <!-- 悪い例：見出しがない -->
   <section>
     <p>内容...</p>
   </section>
   <!-- 見出しがないならdivの方がいいかも -->
   ```

### よくある問題と修正方法

#### 問題 1：div だらけのコード

**症状**：AI が生成したコードが div ばかり

```html
<div class="header">
  <div class="title">タイトル</div>
</div>
<div class="content">
  <div class="post">...</div>
</div>
```

**原因**：AI への指示が不十分

**修正方法**：

```html
<header>
  <h1>タイトル</h1>
</header>
<main>
  <article>...</article>
</main>
```

**AI への追加指示**：

```text
「divではなく、適切なセマンティック要素
（header, main, article, section など）を使って書き直してください」
```

---

#### 問題 2：main が複数ある

**症状**：

```html
<main>
  <article>記事1</article>
</main>
<main>
  <article>記事2</article>
</main>
```

**原因**：main タグのルールを AI が誤解

**修正方法**：

```html
<main>
  <article>記事1</article>
  <article>記事2</article>
</main>
```

**AI への追加指示**：

```text
「mainタグは1ページに1つだけにして、
その中に複数のarticleを入れてください」
```

---

#### 問題 3：article と section の混同

**症状**：単独で意味が通じない内容が article になってる

```html
<article>
  <h2>私のスキル</h2>
  <p>HTML と CSS</p>
</article>
```

**原因**：article の使い方の誤解

**判断基準**：これだけ別のサイトに持っていっても意味が通じる？
→ NO なら section に変更

**修正方法**：

```html
<section>
  <h2>私のスキル</h2>
  <p>HTML と CSS</p>
</section>
```

---

#### 問題 4：不要な div のネスト

**症状**：意味のない div が何重にも重なってる

```html
<div>
  <div>
    <div>
      <p>テキスト</p>
    </div>
  </div>
</div>
```

**原因**：AI が過剰に構造化

**修正方法**：

```html
<p>テキスト</p>
```

または、スタイリングが必要なら：

```html
<div class="container">
  <p>テキスト</p>
</div>
```

**AI への追加指示**：

```text
「不要なdivを削除して、シンプルな構造にしてください」
```

  - 生成されたコードの評価方法

### 重要なポイント

セマンティック HTML は、**コードに意味を与える**！

```html
<!-- Before: 意味が不明 -->
<div>
  <div>タイトル</div>
  <div>本文...</div>
</div>

<!-- After: 意味が明確！ -->
<article>
  <h2>タイトル</h2>
  <p>本文...</p>
</article>
```

**これができると**：

- Google があなたのサイトを正しく理解してくれる 🔍
- スクリーンリーダーで読みやすくなる ♿
- AI が的確なコードを生成してくれる 🤖
- チームメンバーがコードを読みやすくなる 👥

**まるで、コードに名札をつけてあげるみたい！** 🏷️✨

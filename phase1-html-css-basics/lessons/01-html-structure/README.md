## HTML ドキュメントの基本構造

### 最小限の HTML ファイル

これが、最もシンプルな HTML ファイルだよ：

```html
<!DOCTYPE html>
<html lang="ja">
  <head>
    <meta charset="UTF-8" />
    <title>私の最初のページ</title>
  </head>
  <body>
    <h1>Hello, World!</h1>
    <p>これが私の最初のHTMLページです。</p>
  </body>
</html>
```

**たったこれだけ！** でも、各部分には重要な役割があるんだ。

---

### 各部分の役割を理解しよう

#### 1. `<!DOCTYPE html>`

```html
<!DOCTYPE html>
```

- **役割**：これは HTML5 ドキュメントだよ、とブラウザに宣言する
- **位置**：ファイルの一番最初
- **必須度**：⭐⭐⭐⭐⭐（絶対必要！）

**例え話**：お店の看板みたいなもの。「ここは HTML5 のお店です！」って宣言してる。

---

#### 2. `<html>` タグ

```html
<html lang="ja">
  <!-- ここに全ての内容が入る -->
</html>
```

- **役割**：HTML ドキュメント全体を囲む大きな箱
- **lang 属性**：このページの言語を指定（`ja`=日本語、`en`=英語）
- **必須度**：⭐⭐⭐⭐⭐（絶対必要！）

**例え話**：建物全体の外壁。すべてがこの中に入る。

---

#### 3. `<head>` タグ

```html
<head>
  <meta charset="UTF-8" />
  <title>私の最初のページ</title>
</head>
```

- **役割**：ページの「設定情報」を書く場所
- **表示される？**：❌ ブラウザには表示されない
- **何を書く？**：文字コード、タイトル、CSS のリンクなど

**例え話**：Word の「ファイル → プロパティ」みたいなもの。文書の情報を管理する場所。

##### `<meta charset="UTF-8">`

```html
<meta charset="UTF-8" />
```

- **役割**：文字コードの指定（日本語が正しく表示されるために必要）
- **UTF-8**：世界中のあらゆる文字に対応できる文字コード
- **必須度**：⭐⭐⭐⭐⭐（これがないと文字化けする！）

##### `<title>` タグ

```html
<title>私の最初のページ</title>
```

- **役割**：ブラウザのタブに表示されるタイトル
- **表示場所**：ブラウザのタブ、ブックマーク、検索結果
- **必須度**：⭐⭐⭐⭐⭐（SEO にも超重要！）

---

#### 4. `<body>` タグ

```html
<body>
  <h1>Hello, World!</h1>
  <p>これが私の最初のHTMLページです。</p>
</body>
```

- **役割**：実際にブラウザに表示される内容を書く場所
- **表示される？**：✅ ここに書いたものが全部表示される
- **何を書く？**：見出し、段落、画像、リンクなど

**例え話**：Word の本文エリア。ここに書いた内容が実際のページになる！

---

## 完全版の HTML テンプレート

実際のプロジェクトでは、もう少し情報を追加することが多いよ：

```html
<!DOCTYPE html>
<html lang="ja">
  <head>
    <!-- 文字コード（必須） -->
    <meta charset="UTF-8" />

    <!-- ビューポート設定（スマホ対応に必要） -->
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />

    <!-- ページの説明文（検索結果に表示される） -->
    <meta name="description" content="私の素敵なウェブサイト" />

    <!-- ページのタイトル（必須） -->
    <title>私のウェブサイト</title>
  </head>
  <body>
    <!-- ここに実際のコンテンツを書く -->
    <h1>ようこそ！</h1>
    <p>これは私のウェブサイトです。</p>
  </body>
</html>
```

### 追加した要素の説明

#### `<meta name="viewport">`

```html
<meta name="viewport" content="width=device-width, initial-scale=1.0" />
```

- **役割**：スマホやタブレットで正しく表示されるようにする
- **必須度**：⭐⭐⭐⭐（今の時代はほぼ必須）
- **Phase 2 で詳しく学ぶよ！**

#### `<meta name="description">`

```html
<meta name="description" content="ページの説明文" />
```

- **役割**：Google などの検索結果に表示される説明文
- **必須度**：⭐⭐⭐（SEO 対策で重要）

## 🤖 バイブコーディング実践

### AI への指示例

HTML の基本構造を AI に作ってもらう時は、こんな風に指示しよう！

#### ⭕ 良い指示の例

```text
「HTML5の基本構造で、タイトルが『私の趣味』、
言語が日本語のHTMLファイルを作ってください。
bodyには『趣味について』という見出しと、
『私の趣味は読書です』という段落を含めてください。」
```

**ポイント**：

- HTML5 であることを明示
- タイトルを具体的に指定
- 言語を指定
- body 内の内容も具体的に

#### ❌ 曖昧な指示の例

```text
「HTMLファイル作って」
```

**問題点**：

- どんな内容か分からない
- タイトルが何か不明
- 最低限の情報しか生成されない

---

### 生成されたコードの読み方

AI がコードを生成したら、以下のポイントをチェックしよう！

#### ✅ チェックリスト

1. **DOCTYPE 宣言があるか？**

   ```html
   <!DOCTYPE html>
   ```

   → これがないとブラウザが正しく表示できない可能性がある

2. **lang 属性が正しいか？**

   ```html
   <html lang="ja"></html>
   ```

   → 日本語のページなら`lang="ja"`になってるはず

3. **文字コード指定があるか？**

   ```html
   <meta charset="UTF-8" />
   ```

   → これがないと日本語が文字化けする！

4. **title タグがあるか？**

   ```html
   <title>適切なタイトル</title>
   ```

   → 空っぽの title は避けよう

5. **インデント（字下げ）が綺麗か？**

   ```html
   <!-- 良い例：ネストが分かりやすい -->
   <html>
     <head>
       <title>タイトル</title>
     </head>
   </html>
   ```

---

### よくある問題と修正方法

#### 問題 1：日本語が文字化けする

**症状**：ブラウザで開くと「ã"ã‚"ã«ã¡ã¯」みたいになる

**原因**：

- `<meta charset="UTF-8">`がない
- ファイルが UTF-8 で保存されていない

**修正方法**：

```html
<head>
  <!-- これを追加 -->
  <meta charset="UTF-8" />
  <title>タイトル</title>
</head>
```

そして、エディタでファイルを**UTF-8 形式で保存**する。

---

#### 問題 2：タイトルが「localhost」や「file://」になる

**症状**：ブラウザのタブに変なタイトルが表示される

**原因**：`<title>`タグがない、または空っぽ

**修正方法**：

```html
<head>
  <meta charset="UTF-8" />
  <!-- 適切なタイトルを設定 -->
  <title>私のページ</title>
</head>
```

---

#### 問題 3：コードが読みづらい

**症状**：全部が 1 行になってる、インデントがバラバラ

**AI への追加指示**：

```text
「インデントを適切に整えて、読みやすくしてください」
```

または、VS Code なら：

- Windows/Linux：`Shift + Alt + F`
- Mac：`Shift + Option + F`

でコードを自動整形できるよ！

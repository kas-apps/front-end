# Lesson 2: HTML 基本要素 📝

**学習目標**：HTML の基本的な要素を理解し、コンテンツを構造化できるようになる

---

## なぜ HTML 基本要素を学ぶの？

Lesson 1 で「HTML 文書の骨格」を学んだね。今度は**中身（コンテンツ）** を作る方法を学ぼう！

Word/Pages で文書を作る時に：

- 見出しを付けて
- 段落を書いて
- 箇条書きで整理して
- 画像を入れて
- リンクを張る

**HTML でも全く同じことができる！** ✨

この基本要素を使いこなせれば：

- ✅ ブログ記事が書ける
- ✅ 自己紹介ページが作れる
- ✅ 情報を整理して伝えられる

**まるで Word のツールバーを使いこなすみたいな感じ！** 📝

## 見出し：h1〜h6

### 役割と使い方

見出しは、文書の構造を作る超重要な要素！

```html
<h1>これが最も大きな見出し</h1>
<h2>2番目に大きな見出し</h2>
<h3>3番目に大きな見出し</h3>
<h4>4番目に大きな見出し</h4>
<h5>5番目に大きな見出し</h5>
<h6>最も小さな見出し</h6>
```

### 使い分けのルール

- **h1**：ページに 1 つだけ！ページ全体のタイトル
- **h2**：大きなセクションの見出し
- **h3**：h2 の中の小見出し
- **h4〜h6**：さらに細かい階層（あまり使わない）

### 良い例・悪い例

```html
<!-- ⭕ 良い例：階層構造が明確 -->
<h1>フロントエンド入門</h1>

<h2>第1章：HTMLの基礎</h2>
<h3>1.1 HTMLとは</h3>
<h3>1.2 基本構造</h3>

<h2>第2章：CSSの基礎</h2>
<h3>2.1 CSSとは</h3>

<!-- ❌ 悪い例：階層がバラバラ -->
<h1>タイトル</h1>
<h4>いきなり小見出し</h4>
<!-- h2やh3を飛ばしてる！ -->
<h2>あれ？戻ってる</h2>
```

**ポイント**：見出しは順番に使う！h1→h2→h3 の順で、飛ばさない。

## 段落：p

### 役割と使い方

段落は、文章のまとまりを表す基本中の基本！

```html
<p>これが1つの段落です。</p>
<p>これは別の段落です。段落と段落の間には、自動的に余白が入ります。</p>
```

### 複数行の文章

```html
<p>
  段落の中で改行したい時は、 普通に改行しても表示には反映されません。
  ブラウザは自動的に1行にまとめます。
</p>

<!-- 改行したい場合は<br>タグを使う -->
<p>
  1行目<br />
  2行目<br />
  3行目
</p>
```

**Word/Pages との違い**：Enter キーで改行しても、HTML では表示に反映されない。改行したい時は`<br>`タグを使おう！

---

## リスト：ul, ol, li

### 順序なしリスト（ul）

箇条書きリストを作る時に使うよ！

```html
<ul>
  <li>リンゴ</li>
  <li>バナナ</li>
  <li>オレンジ</li>
</ul>
```

**表示結果**：

```text
• リンゴ
• バナナ
• オレンジ
```

### 順序付きリスト（ol）

番号付きリストを作る時に使うよ！

```html
<ol>
  <li>材料を準備する</li>
  <li>鍋に水を入れる</li>
  <li>沸騰させる</li>
</ol>
```

**表示結果**：

```text
1. 材料を準備する
2. 鍋に水を入れる
3. 沸騰させる
```

### ネストしたリスト

リストの中にリストを入れることもできる！

```html
<ul>
  <li>
    フルーツ
    <ul>
      <li>リンゴ</li>
      <li>バナナ</li>
    </ul>
  </li>
  <li>
    野菜
    <ul>
      <li>トマト</li>
      <li>キュウリ</li>
    </ul>
  </li>
</ul>
```

**ポイント**：`<li>`の中に新しい`<ul>`や`<ol>`を入れる！

## リンク：a

### 基本的な使い方

他のページへのリンクを作るよ！

```html
<a href="https://example.com">リンクのテキスト</a>
```

- **href 属性**：リンク先の URL
- **リンクテキスト**：クリックできる部分

### 新しいタブで開く

```html
<a href="https://example.com" target="_blank">新しいタブで開く</a>
```

**target="\_blank"**：リンクを新しいタブで開く

### 同じサイト内へのリンク

```html
<!-- 相対パス -->
<a href="about.html">このサイトのaboutページへ</a>
<a href="/contact.html">トップからのパス</a>

<!-- ページ内リンク -->
<a href="#section1">このページの特定の場所へ</a>
```

### メールリンク

```html
<a href="mailto:info@example.com">メールを送る</a>
```

クリックするとメールアプリが開くよ！

## 画像：img

### 基本的な使い方

```html
<img src="photo.jpg" alt="写真の説明" />
```

- **src 属性**：画像ファイルのパス
- **alt 属性**：画像の説明文（超重要！）

### なぜ alt 属性が重要？

1. **アクセシビリティ**：視覚障害のある人がスクリーンリーダーで内容を理解できる
2. **SEO**：検索エンジンが画像の内容を理解できる
3. **エラー時**：画像が表示できない時に説明文が表示される

```html
<!-- ⭕ 良い例 -->
<img src="cat.jpg" alt="庭で遊ぶ茶色い猫" />

<!-- ❌ 悪い例 -->
<img src="cat.jpg" alt="" />
<!-- 説明がない -->
<img src="cat.jpg" />
<!-- alt属性自体がない -->
```

### サイズの指定

```html
<img src="photo.jpg" alt="写真" width="300" height="200" />
```

**注意**：通常は CSS でサイズを指定する方が良いよ（Phase 1 の後半で学ぶ）！

### 相対パスと絶対パス

```html
<!-- 同じフォルダの画像 -->
<img src="photo.jpg" alt="写真" />

<!-- imagesフォルダ内の画像 -->
<img src="images/photo.jpg" alt="写真" />

<!-- ウェブ上の画像 -->
<img src="https://example.com/photo.jpg" alt="写真" />
```

## テーブル：table

### 基本的な構造

表を作る時に使うよ！

```html
<table>
  <tr>
    <th>名前</th>
    <th>年齢</th>
    <th>職業</th>
  </tr>
  <tr>
    <td>田中太郎</td>
    <td>25</td>
    <td>エンジニア</td>
  </tr>
  <tr>
    <td>鈴木花子</td>
    <td>30</td>
    <td>デザイナー</td>
  </tr>
</table>
```

**要素の説明**：

- `<table>`：表全体を囲む
- `<tr>`：1 行（table row）
- `<th>`：見出しセル（table header）
- `<td>`：データセル（table data）

### より詳細な表

```html
<table>
  <thead>
    <tr>
      <th>商品名</th>
      <th>価格</th>
      <th>在庫</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>りんご</td>
      <td>100円</td>
      <td>在庫あり</td>
    </tr>
    <tr>
      <td>バナナ</td>
      <td>150円</td>
      <td>在庫なし</td>
    </tr>
  </tbody>
</table>
```

- `<thead>`：表のヘッダー部分
- `<tbody>`：表のボディ部分

**ポイント**：thead, tbody を使うと、後で CSS でスタイリングしやすくなる！

## コンテナ要素：div と span

### なぜ div と span が必要？

これまで学んだ要素（h1, p, ul, img など）は、それぞれ「見出し」「段落」「リスト」「画像」という**意味**を持っていたね。

でも、時には **「単にグループ化したい」「スタイリングのために囲みたい」** という場面がある。そんな時に使うのが `<div>` と `<span>` だよ！

**特徴**：

- ✅ 特定の意味を持たない（汎用的なコンテナ）
- ✅ レイアウトやスタイリングのために使う
- ✅ CSS でデザインを適用する時に便利

**例え話**：透明な収納ボックス。中身に意味はないけど、整理整頓やラベル貼りに使える！📦

---

### div：ブロックレベルのコンテナ

`<div>` は、複数の要素をまとめる「箱」として使うよ。

```html
<!-- 複数の要素をグループ化 -->
<div>
  <h2>商品情報</h2>
  <p>価格：1,000円</p>
  <p>在庫あり</p>
</div>
```

**特徴**：

- 改行されて表示される（ブロックレベル要素）
- 幅いっぱいに広がる
- 上下に余白が入る

**よく使う場面**：

- 関連する要素をグループ化
- レイアウトのための区切り
- CSS でスタイルを適用するための囲み

---

### span：インラインレベルのコンテナ

`<span>` は、テキストの一部をグループ化する時に使うよ。

```html
<p>この<span>部分だけ</span>に色を付けたい</p>
```

**特徴**：

- 改行されない（インラインレベル要素）
- テキストの流れの中で使える
- 必要な幅だけ占有

**よく使う場面**：

- テキストの一部に色を付ける
- テキストの一部だけフォントを変える
- アイコンをテキスト内に入れる

---

### div と span の違い

| 項目   | div                    | span                       |
| ------ | ---------------------- | -------------------------- |
| レベル | ブロックレベル         | インラインレベル           |
| 改行   | される                 | されない                   |
| 幅     | 親要素いっぱい         | 内容分だけ                 |
| 用途   | セクションのグループ化 | テキストの一部をグループ化 |

**表示例**：

```html
<!-- divは改行される -->
<div>これは div</div>
<div>別の div</div>

<!-- 表示結果 -->
<!-- これは div -->
<!-- 別の div -->

<!-- spanは改行されない -->
<span>これは span</span>
<span>別の span</span>

<!-- 表示結果 -->
<!-- これは span別の span -->
```

---

### 良い使い方・避けるべき使い方

#### ⭕ 良い使い方

```html
<!-- レイアウトのための囲み -->
<div class="card">
  <h3>商品名</h3>
  <p>説明文</p>
</div>

<!-- テキストの一部にスタイル -->
<p>価格：<span class="price">1,000円</span></p>
```

**ポイント**：

- 意味のある要素（h1, p など）が他にない時に使う
- CSS でスタイリングする目的が明確

---

#### ❌ 避けるべき使い方

```html
<!-- 悪い例：見出しをdivで作ってる -->
<div class="title">ページタイトル</div>
<!-- h1を使うべき！ -->

<!-- 悪い例：段落をdivで作ってる -->
<div>これは文章です</div>
<!-- pを使うべき！ -->
```

**問題点**：

- 検索エンジンが内容を理解できない
- スクリーンリーダーが正しく読み上げられない
- コードが読みづらい

**正しい例**：

```html
<!-- ⭕ 意味のある要素を使う -->
<h1>ページタイトル</h1>
<p>これは文章です</p>

<!-- divは、それらをグループ化する時に使う -->
<div class="header">
  <h1>ページタイトル</h1>
  <p>説明文</p>
</div>
```

---

### 使い分けの判断基準

**質問 1**：これは何？（見出し？段落？リスト？）

- **答えがある** → h1, p, ul などの意味のある要素を使う
- **答えがない** → div か span を検討

**質問 2**：改行が必要？

- **はい** → div（ブロックレベル）
- **いいえ** → span（インラインレベル）

**質問 3**：何のために使う？

- **レイアウト・グループ化** → OK！
- **単なる見出しや段落の代わり** → NG！意味のある要素を使おう

---

### 次のレッスンでもっと良い方法を学ぶよ！

div と span は便利だけど、**「これは何？」という意味が伝わらない**んだ。

例えば：

```html
<div class="header">...</div>
<div class="main">...</div>
<div class="footer">...</div>
```

これだと、全部 div で区別がつかないよね。

**次の Lesson 3 では、もっと意味のある要素**を学ぶよ！

- `<header>`：ヘッダー
- `<main>`：メインコンテンツ
- `<footer>`：フッター
- `<article>`, `<section>`, `<nav>`, `<aside>` などなど

これらを使えば、**コードが自分で説明してくれる**ようになる！🎯

### よくある問題と修正方法

#### 問題 1：見出しの順番が飛んでる

**症状**：h1 の次にいきなり h3

```html
<!-- ❌ 悪い例 -->
<h1>タイトル</h1>
<h3>小見出し</h3>
<!-- h2が抜けてる！ -->
```

**修正方法**：

```html
<!-- ⭕ 良い例 -->
<h1>タイトル</h1>
<h2>セクション</h2>
<h3>小見出し</h3>
```

---

#### 問題 2：リストの構造が壊れてる

**症状**：`<li>`が`<ul>`の外にある

```html
<!-- ❌ 悪い例 -->
<li>項目1</li>
<ul>
  <li>項目2</li>
</ul>
```

**修正方法**：

```html
<!-- ⭕ 良い例 -->
<ul>
  <li>項目1</li>
  <li>項目2</li>
</ul>
```

---

#### 問題 3：画像が表示されない

**症状**：壊れた画像アイコンが表示される

**原因**：

- ファイルパスが間違っている
- 画像ファイルが存在しない

**修正方法**：

```html
<!-- ファイルパスを確認 -->
<img src="images/photo.jpg" alt="写真" />

<!-- 画像ファイルが同じフォルダにあるか確認 -->
```

**AI への追加指示**：

```text
「画像ファイルが『images』フォルダの中にあることを前提に、
正しいパスを使ってください」
```

---

#### 問題 4：リンクが機能しない

**症状**：クリックしても何も起こらない

**原因**：href 属性が空っぽ、または`#`だけ

```html
<!-- ❌ 悪い例 -->
<a href="#">リンク</a>
<a href="">リンク</a>
```

**修正方法**：

```html
<!-- ⭕ 良い例 -->
<a href="https://example.com">リンク</a>
<a href="page.html">リンク</a>
```

---

#### 問題 5：テーブルのレイアウトが崩れる

**症状**：表の列がずれてる

**原因**：各行の`<td>`や`<th>`の数が揃ってない

```html
<!-- ❌ 悪い例 -->
<table>
  <tr>
    <th>名前</th>
    <th>年齢</th>
  </tr>
  <tr>
    <td>田中</td>
    <td>25</td>
    <td>エンジニア</td>
    <!-- 3つ目がある！ -->
  </tr>
</table>
```

**修正方法**：

```html
<!-- ⭕ 良い例 -->
<table>
  <tr>
    <th>名前</th>
    <th>年齢</th>
    <th>職業</th>
    <!-- ヘッダーに追加 -->
  </tr>
  <tr>
    <td>田中</td>
    <td>25</td>
    <td>エンジニア</td>
  </tr>
</table>
```

## 📝 まとめ

このレッスンで学んだこと：

- ✅ **見出し（h1〜h6）** で文書を構造化

  - h1 はページに 1 つ、階層を守る

- ✅ **段落（p）** で文章を書く

  - 改行は`<br>`タグ

- ✅ **リスト（ul, ol）** で情報を整理

  - ul：箇条書き、ol：番号付き

- ✅ **リンク（a）** で他のページに繋ぐ

  - href 属性が超重要

- ✅ **画像（img）** でビジュアルを追加

  - alt 属性を必ず付ける

- ✅ **テーブル（table）** でデータを整理

  - thead, tbody で構造を明確に

- ✅ **コンテナ要素（div, span）** を理解した
  - div：ブロックレベルのグループ化
  - span：インラインレベルのグループ化
  - 意味がある要素を優先、グループ化にはこれらを使う

### Word/Pages との対応

| Word/Pages でやること | HTML でやること |
| --------------------- | --------------- |
| 見出しスタイルを適用  | h1〜h6 タグ     |
| 段落を書く            | p タグ          |
| 箇条書き              | ul + li         |
| 番号付きリスト        | ol + li         |
| ハイパーリンク挿入    | a タグ          |
| 画像挿入              | img タグ        |
| 表を挿入              | table タグ      |

**やってることは本質的に同じ！** ただ、書き方が違うだけ！

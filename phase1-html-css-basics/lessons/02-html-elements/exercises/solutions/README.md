### 応用編

#### [02-04.html](02-04.html) - 演習 4 の解答：商品一覧ページ

**演習内容**：

- h1：「商品一覧」
- 3 つの商品について：
  - h2：商品名
  - img：商品画像
  - p：商品説明
  - リンク：「詳細を見る」

**ポイント**：

- ✅ 画像に架空のパス（`images/xxx.jpg`）を使用
- ✅ 画像に適切な alt 属性を設定
- ✅ 各商品に詳細ページへのリンクを配置
- ✅ コメントで画像が架空のものであることを説明

**⚠️ 注意**：

画像ファイルは架空のパスです。実際に使用する場合は：

1. フリー素材サイト（Unsplash、Pexels など）から画像をダウンロード
2. プロジェクトのルートに `images` フォルダを作成
3. ダウンロードした画像を `images` フォルダに保存
4. ファイル名を解答例と同じにする（`laptop.jpg`, `earphones.jpg`, `smartwatch.jpg`）

## 🎯 学習のポイント

これらの解答例を見る時は、以下のポイントに注目しよう：

### 1. 見出しの階層構造

- h1 はページに 1 つだけ
- h2, h3 は順番に使う（飛ばさない）
- 文書の構造が一目で分かる

### 2. リストの使い分け

- **ul（箇条書き）**：順序が関係ない項目
  - 例：好きなもの、材料リスト、魅力のポイント
- **ol（番号付きリスト）**：順序が重要な項目
  - 例：作り方の手順、学習ステップ

### 3. 画像の扱い

- **src 属性**：画像のパスを指定
- **alt 属性**：画像の説明を必ず記述
- この解答例では架空のパス（`images/xxx.jpg`）を使用

### 4. リンクの作成

- **a タグ**：ハイパーリンクを作成
- **href 属性**：リンク先の URL を指定
- 外部サイトへは完全な URL（`https://...`）を使う

### 5. テーブルの構造

- **thead**：ヘッダー行（見出し）
- **tbody**：データ行
- **th**：見出しセル
- **td**：データセル
- 各行の列数を揃える

### 6. コメントの活用

- 各セクションの役割を説明
- 架空の画像ファイルについて注意書き
- 実務でも同様にコメントを書くと分かりやすい

# Phase 2: HTML/CSS 発展編 🚀

**目標**：インタラクティブで動的なレイアウトを作れるようになる

---

## この Phase で学ぶこと

Phase 1 では、静的なドキュメント（ブログ記事や自己紹介ページ）を作る方法を学んだね！

Phase 2 では、その知識をベースに、**もっと動的でインタラクティブな Web ページ**を作る方法を学んでいくよ！

### できるようになること

- ユーザー入力を受け取るフォームが作れる
- Flexbox で柔軟な 1 次元レイアウトが作れる
- Grid で複雑な 2 次元レイアウトが作れる
- スマホ・タブレット・PC に対応したレスポンシブデザインができる
- アニメーションで動きのあるページが作れる

**これだけできれば、本格的な Web サイトやアプリの UI が作れるよ！** 💪

## 🚀 Phase 2 の進め方

### 推奨ステップ

1. **Lesson 1 から順番に進める**

   - Flexbox と Grid は独立しているけど、フォームは先に学んでおくと便利
   - レスポンシブデザインは Flexbox/Grid の知識が前提
   - アニメーションは最後に学ぶと効果的

2. **実際に手を動かす**

   - サンプルコードを自分で書いてみる
   - AI に指示して、生成されたコードを観察する
   - ブラウザの開発者ツールで動作を確認する

3. **演習問題に挑戦する**

   - 各レッスンの演習問題を必ず解く
   - わからなければ AI に相談しながらで OK
   - 解答例と自分のコードを比較する

4. **ミニプロジェクトで実践**
   - Phase 2 の最後に、学んだことを全部使ったプロジェクトに挑戦
   - お問い合わせフォーム、ランディングページ、ギャラリーサイトなど

## 🤖 バイブコーダー向けポイント

### Phase 2 での AI の使い方

Phase 2 では、より複雑なレイアウトを AI と一緒に作っていくよ：

#### 1. レイアウトの指示を具体的に

```
良い指示の例：
「flexboxを使って、3つのカードを横並びに配置してください。
各カードは等幅で、間隔は20px、モバイルでは縦並びにしてください」

悪い指示の例：
「カードをいい感じに並べて」
```

#### 2. レスポンシブ対応を明示

```
良い指示の例：
「PCでは3カラム、タブレットでは2カラム、
スマホでは1カラムになるグリッドレイアウトを作ってください」

悪い指示の例：
「スマホでも見られるようにして」
```

#### 3. アニメーションの詳細を指定

```
良い指示の例：
「ボタンにホバーした時、0.3秒かけて背景色が青から緑に
変わるアニメーションを追加してください」

悪い指示の例：
「ボタンに動きをつけて」
```

### チェックポイント

AI が生成したコードを見る時、Phase 2 ではこんなポイントをチェックしよう：

- ✅ **フォームのアクセシビリティ**

  - label と input が正しく関連付けられているか
  - name 属性が設定されているか
  - 適切なバリデーションがあるか

- ✅ **Flexbox/Grid の適切な使い分け**

  - 1 次元レイアウトには Flexbox
  - 2 次元レイアウトには Grid
  - 不必要に複雑になっていないか

- ✅ **レスポンシブ対応**

  - メディアクエリが適切に設定されているか
  - ブレークポイントが適切か
  - モバイルファーストになっているか

- ✅ **アニメーションのパフォーマンス**
  - transform と opacity を使っているか（パフォーマンス良好）
  - width や height のアニメーションは避けているか（重い）
  - アニメーション時間が適切か（0.2-0.5 秒程度）

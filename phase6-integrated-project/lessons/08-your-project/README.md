# Lesson 8：あなたのプロジェクトを作ろう！

**所要時間**：自由（20-40時間を推奨）
**難易度**：⭐⭐⭐⭐⭐

## 🎯 学習目標

Lesson 1-7で学んだパターンを応用して、あなた自身のオリジナルプロジェクトを作り上げるよ！

**目標**：

- 自分のアイデアを要件定義に落とし込む
- AIと協働しながら仕様を設計する
- Lesson 1-7のパターンを応用して実装する
- 実用レベルのWebアプリケーションを完成させる

**完成イメージ**：

- 自分が本当に使いたいアプリ
- 友達や家族に見せて自慢できるレベル
- ポートフォリオに載せられる作品
- コード規模：600-1000行

## 💡 なぜ自分のプロジェクトを作るのか（Why）

### チュートリアルと自分のプロジェクトの違い

```text
❌ チュートリアルだけでは：
- 「作らされている」感覚
- 完成したときの達成感が薄い
- 自分のものという実感が湧かない
- 応用力が身につきにくい
```

```text
✅ 自分のプロジェクトでは：
- 「作りたい」という強いモチベーション
- 完成したときの達成感が大きい
- 自分の作品として誇れる
- 応用力が自然と身につく
```

### 学習効果の違い

```text
Lesson 1-7（TODOアプリ）：
「TODOアプリの作り方」を学ぶ

Lesson 8（あなたのプロジェクト）：
「Webアプリの作り方」を学ぶ

→ パターンを応用する力が身につく！
```

## 🚀 ステップ1：アイデアを決める（1-2時間）

### アイデアの見つけ方

#### 方法1：日常の不便を解決する

```text
質問：「日常生活で、こんなアプリがあったらいいのに」と思うことは？

例：
- レシピを管理したい → レシピ管理アプリ
- 読んだ本を記録したい → 読書記録アプリ
- 支出を記録したい → 支出管理アプリ
- 習慣を継続したい → 習慣トラッカー
- メモを整理したい → メモアプリ
```

#### 方法2：既存のアプリを参考にする

```text
質問：「このアプリ、自分でも作れそう」と思うものは？

例：
- Trello → カンバン型タスク管理
- Notion → メモ＋データベース
- Habitica → ゲーム化した習慣トラッカー
- Goodreads → 読書管理SNS
- Splitwise → 割り勘計算アプリ
```

#### 方法3：プロジェクトテンプレートから選ぶ

このレッスンでは、5つのプロジェクトテンプレートを用意しているよ！

1. **レシピ管理アプリ** 👨‍🍳
   - お気に入りのレシピを保存・管理
   - 材料、手順、調理時間、難易度を記録
   - ジャンルやタグでフィルター

2. **読書記録アプリ** 📚
   - 読んだ本を記録
   - 評価、感想、読了日を記録
   - 読書目標を設定して進捗管理

3. **支出管理アプリ** 💰
   - 日々の支出を記録
   - カテゴリ別に集計
   - 月間予算を設定して管理

4. **習慣トラッカー** ✅
   - 継続したい習慣を記録
   - カレンダーで達成状況を可視化
   - ストリーク（連続達成日数）を表示

5. **メモアプリ** 📝
   - メモを作成・編集・削除
   - タグやカテゴリで整理
   - 検索機能で素早く見つける

それぞれのメリット・デメリットと、
初心者が実装する際の難易度を教えてください。」

## 📋 ステップ2：要件定義を作成（2-3時間）

Lesson 1で学んだパターンを使って、要件定義を作成しよう！

### 使用するテンプレート

👉 [your-requirements-template.md](../01-requirements/your-requirements-template.md)

### 記入のポイント

1. **プロジェクト概要**
   - 何を作るか：1-2文で簡潔に
   - 誰が使うか：ターゲットユーザーを明確に
   - どんな問題を解決するか：3-5個の箇条書き

2. **ユーザーストーリー**
   - 最低3つ、できれば5-7つ書こう
   - 「ユーザーとして、〇〇したい、なぜなら〇〇」の形式

3. **機能要件**
   - 必須機能：5-7個（これがないとアプリが成り立たない）
   - 拡張機能：3-5個（あると便利）
   - オプション機能：自由（Phase 6では実装しない）

### AIとの協働

```text
【AIへの質問例】

「レシピ管理アプリを作りたいです。

【背景】
- 料理が好きで、いろいろなレシピを試している
- レシピをメモ帳に書いているが、見つけにくい
- レシピを整理して、素早く検索したい

【質問】
レシピ管理アプリに必要な機能を、
必須機能と拡張機能に分けて提案してください。

また、データ構造（レシピオブジェクト）の
プロパティ案も教えてください。」
```

## 📐 ステップ3：仕様書を作成（2-3時間）

Lesson 2で学んだパターンを使って、技術仕様書を作成しよう！

### 使用するテンプレート

👉 [your-spec-template.md](../02-specification/your-spec-template.md)

### 記入のポイント

1. **データ構造**
   - メインオブジェクトのプロパティを定義
   - データ型、必須/任意、デフォルト値、制約を明記

2. **画面構成**
   - HTML構造を設計
   - 要素のID/クラス名を決める

3. **機能仕様**
   - 各機能の処理フローを書く
   - 関数名と役割を定義

4. **LocalStorage仕様**
   - キー名を決める（アプリ名-データ名）
   - 保存タイミングを決める

### AIとの協働

```text
【AIへの質問例】

「以下の要件を実現するために、データ構造を設計したいです。

【要件】
[Lesson 2で作成した要件定義を貼り付け]

JavaScriptのオブジェクト構造を、
具体例と一緒に提案してください。

また、HTML要素のID/クラス名の命名案も
提案してください。」
```

## 🎨 ステップ4：UIデザインとプロトタイプ（3-4時間）

Lesson 3で学んだパターンを使って、UIを設計しよう！

### やること

1. **ワイヤーフレームを作成**
   - テキストベースのレイアウト図
   - カード要素の詳細設計

2. **静的プロトタイプを作成**
   - HTML/CSSのみ（JavaScriptなし）
   - Tailwind CSSでスタイリング
   - レスポンシブ対応

### AIとの協働

```text
【AIへの質問例】

「以下のワイヤーフレームに基づいて、
Tailwind CSSを使ったHTMLを作成してください。

[ワイヤーフレームを貼り付け]

【要件】
- レスポンシブデザイン（モバイル1カラム、タブレット2カラム、デスクトップ3カラム）
- カードにホバーエフェクト
- まずは静的なHTML/CSSのみ（JavaScriptなし）

よろしくお願いします！」
```

## 💻 ステップ5-8：実装（12-16時間）

Lesson 4-7で学んだパターンを応用して、実装していこう！

### ステップ5：基本実装（3-4時間）

**実装する機能**：

- データの追加
- データの一覧表示
- データの削除
- 統計表示（必要に応じて）

**目標行数**：約150-250行

### ステップ6：機能拡張（4-5時間）

**実装する機能**：

- データの編集
- フィルター機能
- ソート機能
- 検索機能

**目標行数**：約400-500行

### ステップ7：永続化とUI改善（3-4時間）

**実装する機能**：

- LocalStorageでのデータ永続化
- ローディング表示
- トースト通知
- アニメーション
- エラーハンドリング

**目標行数**：約500-600行

### ステップ8：リファクタリング（2-3時間）

**改善する観点**：

- コードの整理（関数分割、命名規則）
- パフォーマンス最適化
- アクセシビリティ改善
- ブラウザ互換性確認

**目標行数**：約600-800行（整理されて読みやすい）

## 📁 プロジェクトテンプレート

### 1. レシピ管理アプリ 👨‍🍳

👉 [レシピ管理アプリのアイデア](templates/01-recipe-app/project-idea.md)

**概要**：
お気に入りのレシピを保存・管理できるアプリ

**主な機能**：

- レシピの追加（タイトル、材料、手順、調理時間、難易度、ジャンル）
- レシピの一覧表示（カード形式）
- レシピの編集・削除
- ジャンルでフィルター（和食、洋食、中華など）

### 2. 読書記録アプリ 📚

👉 [読書記録アプリのアイデア](templates/02-reading-log/project-idea.md)

読んだ本を記録し、読書目標を管理できるアプリ

- 本の記録（タイトル、著者、評価、感想、読了日、ジャンル）
- 本の一覧表示（カード形式）

### 3. 支出管理アプリ 💰

👉 [支出管理アプリのアイデア](templates/03-expense-tracker/project-idea.md)

**概要**：
日々の支出を記録し、カテゴリ別に集計できるアプリ

**主な機能**：

- 支出の記録（金額、カテゴリ、日付、メモ）
- 支出の一覧表示（カード形式）
- 支出の編集・削除
- カテゴリでフィルター（食費、交通費、娯楽など）
- 日付でソート
- メモで検索
- 月間集計（カテゴリ別の合計金額）

### 4. 習慣トラッカー ✅

👉 [習慣トラッカーのアイデア](templates/04-habit-tracker/project-idea.md)

**概要**：
継続したい習慣を記録し、達成状況を可視化できるアプリ

**主な機能**：

- 習慣の追加（名前、目標頻度、カテゴリ）
- 習慣の一覧表示（カード形式）
- 習慣の編集・削除
- 日々の達成記録（チェックイン）
- カテゴリでフィルター（健康、勉強、趣味など）
- ストリーク（連続達成日数）の表示
- 週間・月間の達成率

### 5. メモアプリ 📝

👉 [メモアプリのアイデア](templates/05-memo-app/project-idea.md)

**概要**：
メモを作成・整理できるシンプルなアプリ

**主な機能**：

- メモの追加（タイトル、内容、タグ）
- メモの一覧表示（カード形式）
- メモの編集・削除
- タグでフィルター
- 作成日・更新日でソート
- タイトルや内容で検索
- ピン留め機能（重要なメモを上部に固定）

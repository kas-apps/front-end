## なぜ要件定義が重要か（Why）

プログラミングを学び始めた人が陥りがちな罠、それは **「いきなりコードを書き始めてしまう」** こと！

```text
よくある失敗パターン：
「TODOアプリを作ろう！」
↓
いきなりコードを書き始める
↓
「あれ、この機能も欲しい...」「あれ、こっちの機能も...」
↓
コードがぐちゃぐちゃに...
↓
最初からやり直し 😭
```

実は、**良いプロダクトを作るには、コードを書く前の準備が超重要**なんだ！

### 要件定義とは？

要件定義とは、**「何を作るか」を明確にすること**だよ。

- どんな機能が必要？
- 誰が使う？
- どんな問題を解決する？
- 必須の機能は？あったらいいな機能は？

これらを最初に整理しておくことで、**迷わず開発を進められる**し、**無駄なコードを書かなくて済む**んだ！

### AI時代の要件定義

従来は、要件定義には専門知識が必要だったけど、**AI時代は違う！**

AIと対話しながら、以下のようなことができるよ：

- 「こんなアプリを作りたい」というアイデアをAIに相談
- AIから機能の提案をもらう
- 実装の難易度をAIに聞く
- 優先順位をAIと一緒に考える

**AIは最高のブレインストーミングパートナー！** 🤖

## このステップで行うこと（What）

Lesson 1では、以下の流れで要件定義を進めていくよ：

### ステップ1：アイデアの明確化

作りたいものを大まかに決める：

- 何のアプリ？（TODO、レシピ、家計簿など）
- どんな人が使う？（自分、友達、不特定多数など）
- どんな問題を解決する？（タスク管理、料理の記録など）

### ステップ2：AIとのブレインストーミング

AIに相談して、機能のアイデアを広げる：

- どんな機能があると便利？
- 似たアプリにはどんな機能がある？
- 実装が簡単な機能は？難しい機能は？

### ステップ3：ユーザーストーリーの作成

「〜として、〜したい、なぜなら〜」という形式でストーリーを書く：

```text
例：
ユーザーとして、タスクに優先度を設定したい、
なぜなら重要なタスクを見逃さないようにするため
```

### ステップ4：機能の優先順位付け

機能を以下のように分類する：

- **必須機能（Must Have）**：これがないとアプリとして成り立たない
- **拡張機能（Should Have）**：あると便利だけど、なくても動く
- **オプション機能（Nice to Have）**：余裕があれば実装したい

### ステップ5：要件定義書の作成

整理した内容をマークダウンで文書化する

実際に、カード型TODOアプリの要件定義を進めてみよう！

```text

誰が使う？
→ 自分自身（個人のタスク管理）

どんな問題を解決する？

→ フィルターやソートなどの高度な機能
```

## 🤖 AIとの協働（最重要セクション）

### AIへの指示例

要件定義フェーズでは、AIに「提案」や「アドバイス」を求めることが多いよ。

#### ❌ 悪い指示の例

```text
「TODOアプリを作りたいです。」
```

**何が悪い？**

- 作りたいものが漠然としすぎている
- AIがどう提案すればいいか分からない
- 背景や目的が伝わっていない

#### ✅ 良い指示の例

```text
「中規模のTODOアプリを作りたいです。

【背景】
- Phase 3でシンプルなリスト形式のTODOアプリを作りました
- それより高機能で、実用的なものを作りたいです
- カード形式で視覚的に見やすくしたいです

【質問】
どんな機能があると実用的なTODOアプリになりますか？
必須機能と拡張機能に分けて提案してください。
また、それぞれの実装難易度も教えてください。」
```

**何が良い？**

- 背景（Phase 3からのステップアップ）が明確
- 求める方向性（カード形式、高機能）が伝わっている
- 具体的な質問（必須 vs 拡張、難易度）をしている

### AIとの対話の進め方

要件定義フェーズでは、以下のようにAIと対話を重ねていくよ：

#### 対話例1：機能の提案をもらう

```text
あなた：
「中規模のTODOアプリを作りたいです。
カード形式で表示し、Phase 3のシンプルなリスト形式より
高機能にしたいです。どんな機能があると実用的になりますか？
必須機能と拡張機能に分けて提案してください。」

AI：
【必須機能】
1. タスクの追加（タイトル、説明を入力）
2. タスクの一覧表示（カード形式）
3. タスクの完了/未完了の切り替え
4. タスクの削除
5. タスクの統計表示（全体、完了、未完了の数）

【拡張機能】
1. タスクの編集（追加後に内容を修正できる）
2. 優先度の設定（高・中・低で色分け）
3. カテゴリ分け（仕事、プライベート、買い物など）
4. 締切日の設定
5. フィルター機能（完了/未完了、カテゴリ別）
6. ソート機能（優先度順、締切日順など）
7. 検索機能（タイトルや説明で検索）
8. データの永続化（LocalStorage）

【実装難易度】
- 必須機能：初級〜中級（Phase 3の知識でOK）
- 拡張機能1-4：中級（Phase 3-4の知識）
- 拡張機能5-8：中級〜上級（Phase 4-5の知識）
```

#### 対話例2：機能の詳細を掘り下げる

```text
あなた：
「優先度の設定機能について詳しく教えてください。
どのように実装すればいいですか？」

AI：
優先度機能は以下のように実装できます：

【データ構造】
タスクオブジェクトに priority プロパティを追加：
{
  id: 1,
  title: "レポート提出",
  priority: "high" // "high", "medium", "low"
}

【UI表示】
- 高優先度：赤色のバッジ
- 中優先度：黄色のバッジ
- 低優先度：緑色のバッジ

【実装方法】
1. フォームに優先度選択のselectボックスを追加
2. タスク追加時に選択された優先度を保存
3. カード表示時に優先度に応じた色のバッジを表示
4. Tailwind CSSでbg-red-500, bg-yellow-500, bg-green-500を使用
```

#### 対話例3：優先順位を相談する

```text
あなた：
「提案いただいた拡張機能について、実装の優先順位を付けたいです。
重要度と難易度の2軸で評価してください。」

AI：
優先順位マトリクス：

【高重要度 × 低難易度】→ 最優先で実装
1. 優先度の設定（重要度: 高、難易度: 低）
2. カテゴリ分け（重要度: 高、難易度: 低）
3. データの永続化（重要度: 高、難易度: 中）

【高重要度 × 高難易度】→ 次に実装
4. タスクの編集（重要度: 高、難易度: 中）
5. フィルター機能（重要度: 高、難易度: 中）

【中重要度 × 低難易度】→ 余裕があれば実装
6. 締切日の設定（重要度: 中、難易度: 低）
7. ソート機能（重要度: 中、難易度: 中）

【低重要度 × 高難易度】→ 後回し
8. 検索機能（重要度: 中、難易度: 中）

推奨順序：1 → 2 → 3 → 4 → 5 → 6 → 7 → 8
```

### 生成された提案のチェックポイント

AIから機能提案をもらったら、以下をチェックしよう：

- ✅ **実装可能性**

  - 自分の現在のスキルレベルで実装できるか？
  - Phase 0-5で学んだ技術でカバーできるか？
  - 新しく学ぶ必要がある技術はあるか？

- ✅ **必須 vs 拡張の分類**

  - 本当に必須？なくてもアプリは動く？
  - 拡張機能を必須にしすぎていないか？
  - MVP（Minimum Viable Product）として成り立つか？

- ✅ **優先順位**

  - 重要度が高く、難易度が低いものから実装できているか？
  - 難易度が高いものばかり最初に実装しようとしていないか？

- ✅ **具体性**

  - 機能が具体的に説明されているか？
  - 実装方法のヒントが含まれているか？
  - データ構造やUIの例が示されているか？

## よくある問題と解決方法

### 問題1：機能が多すぎて収拾がつかない

**症状**：

- AIに提案してもらったら、機能が20個以上に...
- 全部実装しようとすると、いつ終わるか分からない

**解決方法**：

```text
1. 「これがないとアプリとして成り立たない」機能だけを選ぶ
2. 必須機能は5-7個程度に絞る
3. 拡張機能は「動いてから追加」する

MVPの原則：
Minimum（最小限） Viable（動作する） Product（製品）
まずは最小限の機能で動くものを作ろう！
```

### 問題2：機能が抽象的すぎる

**症状**：

- 「便利な機能が欲しい」「使いやすくしたい」など抽象的
- 実装のイメージが湧かない

**解決方法**：

```text
AIに「具体例を教えてください」と聞こう：

悪い例：「使いやすい機能」
↓
良い例：「タスクをドラッグ&ドロップで並び替えられる機能」

AIに具体化してもらおう！
```

### 問題3：自分のスキルで実装できるか不安

**症状**：

- AIが提案した機能、難しそう...
- 実装方法が全く想像できない

**解決方法**：

```text
AIに難易度を聞こう：

「この機能は、Phase 0-5で学んだ知識で実装できますか？
もし新しい技術が必要なら、何を学ぶ必要があるか教えてください。」

難しすぎる場合は、シンプルな代替案をAIに提案してもらおう！
```

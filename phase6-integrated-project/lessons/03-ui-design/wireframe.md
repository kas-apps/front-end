# カード型TODOアプリ - ワイヤーフレーム

**作成日**：2025-12-01
**プロジェクト名**：Card Todo

---

## ワイヤーフレームとは

ワイヤーフレームは、アプリの「設計図」だよ！
細かいデザインや色は後回しにして、「どこに何を配置するか」を決めるんだ。

**メリット**：

- デザインの迷走を防ぐ
- AIへの指示がより具体的になる
- 実装前に全体像が見える
- チーム開発の場合、共通認識を持てる

## タスクカードの詳細設計

### 未完了タスクのカード

```text
┌─────────────────────────────────────────┐
│ 🔲 Phase 6の教材を完成させる          [高] │  ← タイトル + 優先度バッジ
├─────────────────────────────────────────┤
│ Lesson 1-8とfinal-projectを作成する。     │  ← 説明（2行まで）
│ 学習者がAIと協働しながら...                 │
├─────────────────────────────────────────┤
│ 💼 仕事                                  │  ← カテゴリアイコン
│ 📅 締切: 2025-12-15                      │  ← 締切日
├─────────────────────────────────────────┤
│ [編集] [削除]                            │  ← アクションボタン
└─────────────────────────────────────────┘
```

```text

│ ✅ ~~Lesson 1の要件定義を作成~~    [高]    │  ← 取り消し線
├─────────────────────────────────────────┤
│ 要件定義書のテンプレートと事例を作成          │  ← 半透明表示

├─────────────────────────────────────────┤
│ [編集] [削除]                            │

```

## カード要素の詳細仕様

### 優先度バッジ

```text
高：  [高]  ← 赤色背景、白文字
中：  [中]  ← 黄色背景、黒文字
低：  [低]  ← 緑色背景、白文字
```

### カテゴリアイコン

```text
仕事：        💼 仕事
プライベート： 🏠 プライベート
買い物：      🛒 買い物
```

### チェックボックス

```text
未完了： ☐ （空のボックス）
完了：   ☑ （チェック済み）
```

## AIへの指示に使えるワイヤーフレーム

このワイヤーフレームをAIに渡すときは、こんな感じで伝えよう：

```text
【AIへの指示例】

「以下のワイヤーフレームに基づいて、Tailwind CSSを使った
HTMLを作成してください。

[ワイヤーフレームのASCIIアートを貼り付け]

【要件】
- レスポンシブデザイン（モバイル1カラム、タブレット2カラム、デスクトップ3カラム）
- 優先度バッジは色分け（高=赤、中=黄、低=緑）
- カードにホバーエフェクト（影が濃くなる）
- まずは静的なHTML/CSSのみ（JavaScriptなし）

よろしくお願いします！」
```
